# Mission & Success Contract

## Mission Statement
Sentinel exists to make RTI commissioning structured, testable, and efficient by turning a project's `.apex` file into a live testing environment with a user interface for executing tests and a diagnostics interface for monitoring results, regenerating the test environment after project updates, and guiding troubleshooting.

## Explicit Non-Goals
1. Replacing the RTI system or acting as the live control interface for end users.
2. Prioritizing visual polish, fancy graphics, or unnecessary complexity over clarity and usefulness during commissioning.
3. Depending on manual recreation of project structure when that structure can be generated from the `.apex` file.

## Success Criteria
1. Sentinel provides a user interface and a diagnostics interface that work together during RTI commissioning.
2. The user interface supports both event testing and device testing in a way that matches the needs of live commissioning work.
3. The diagnostics interface helps monitor testing progress, supports regeneration after updated `.apex` uploads, and provides useful troubleshooting context when failures occur.
4. Testing can continue across updated `.apex` files without losing prior test results or forcing the testing process to restart from scratch.
5. Sentinel helps commissioning move faster and with more confidence by making tests clear, repeatable, and reviewable.
